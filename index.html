Understanding The Steps of Compilation

What is GCC?

The GNU Compiler Collection (GCC) is a versatile and widely used optimizing compiler that supports various programming languages, operating systems, and hardware architectures. It's distributed as free software under the GNU General Public License and is a fundamental component of the GNU toolchain. Originally known as the GNU C Compiler, GCC expanded to support multiple languages, including C++, Fortran, Ada, and more. It's a pivotal tool in the world of free software and has been crucial in promoting its growth. GCC has been ported to numerous platforms and is the standard compiler for many Unix-like operating systems, including Linux distributions. While some have switched to alternatives like Clang, GCC remains a powerful choice for compiling code on various platforms, including Windows, Android, and more.

Prepocessor

The preprocessor in C and C++ is a crucial tool that prepares the source code for compilation by handling tasks like file inclusion, macro expansion, conditional compilation, and comments removal. It processes directives like `#include` to include header files, `#define` to create macros, and `#ifdef` for conditional code inclusion. For instance, you can run the preprocessor in GCC (GNU Compiler Collection) using the `-E` option. For example, to preprocess a file named `myFile.c` and output the result to a file `preprocessedMyFile.c`, you can use the following command: gcc -E myFile.c -o preprocessedMyFile.c This command instructs GCC to only run the preprocessor stage, and the preprocessed code will be saved in the `preprocessedMyFile.c` file, making it easier to inspect and debug.

Compiler

To translate the preprocessed code from `myFile.c` to assembly language in `compiledMyFile.s`, you can use the `gcc` command with the `-S` option. For example: gcc -S myFile.c This command instructs the compiler to generate the assembly code from the preprocessed file. In the resulting `compiledMyFile.s` file, you will find the assembly code for your program, including any calls to external functions. In this stage, the preprocessed source code is compiled into assembly code, which is a low-level representation of the program.

Assembler

To convert the assembly code from `compiledMyFile.s` into machine language instructions and produce an object file named `assembledMyFile.o`, you can use the `as` command as follows: as compiledMyFile.s -o assembledMyFile.o Executing this command generates the `assembledMyFile.o` object file, which contains the machine instructions for the program and includes an undefined reference to it's functions.

Linker

This is the final compilation stage for the "Hello World!" program, where object files are linked to create the executable file. The executable relies on various external resources, such as system functions and C runtime libraries.

The command for linking files:

gcc file1.o file2.o -o output_executable

Here's what each part of the command does:

gcc: This is the command to invoke the GCC compiler and linker.

file1.o and file2.o: These are the object files you want to link together. Object files typically have a .o extension and contain compiled code.

-o output_executable: This part specifies the name of the output executable you want to create. You can replace output_executable with your desired name.

After running this command, GCC will link the specified object files and produce the output_executable file, which you can run to execute your program.

Conclusion

Compiling is a crucial part of software development; understanding how it works is essential for any programmer. GCC simplifies this process by providing comprehensive tools to compile code written in various programming languages. By following the steps we've outlined, you can see how your source code is transformed into an executable program that can be run on your computer.
